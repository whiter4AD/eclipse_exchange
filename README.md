# 🌑 Eclipse Exchange

Telegram Mini App для обмена USDT ⇄ RUB по фиксированному курсу, с ботом, личным кабинетом пользователя и админ-панелью.

---

## Возможности

### Для пользователей
- 🔄 **Продажа USDT → ₽** по фиксированному курсу (79 ₽/USDT)
- 💰 **Покупка USDT за ₽** по фиксированному курсу (88 ₽/USDT)
- 💳 **Пополнение баланса** через чек Crypto Bot / xRocket
- 📱 **Пополнение SIM** — баланс телефона по отдельному курсу
- 📷 **Оплата по QR** — с прикреплением фото QR-кода
- 📤 **Вывод USDT** с баланса (чек в личные сообщения)
- ⭐ **Отзывы** — оставляются только после завершённой сделки, без накруток
- 👤 Личный профиль с аватаркой, статистикой и историей сделок
- 📄 Обязательное принятие пользовательского соглашения при первом запуске

### Для администратора
- 📊 Общая статистика (пользователи, заявки, оборот в ₽ и USDT)
- 👥 Список пользователей с поиском, сортировкой, балансом и блокировкой
- ⭐ Просмотр и удаление отзывов
- 📢 Рассылка сообщений всем пользователям (текст и/или фото, с Markdown)
- ✅ Подтверждение/отклонение заявок прямо в Telegram через бота

## Переменные окружения

Указываются в Railway → Variables:

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от [@BotFather](https://t.me/BotFather) |
| `DATABASE_URL` | Строка подключения к PostgreSQL (создаётся автоматически при добавлении БД в Railway) |
| `WEBAPP_URL` | Публичный URL сервиса (домен Railway) |
| `MANAGER_CHAT_ID` | Telegram ID администратора/менеджера — получает уведомления о заявках и имеет доступ к `/admin` |

---

## Деплой на Railway

1. Создать репозиторий на GitHub с этой структурой файлов
2. В Railway: **New Project → Deploy from GitHub repo**
3. Добавить PostgreSQL: **+ New → Database → Add PostgreSQL** (переменная `DATABASE_URL` создастся автоматически)
4. Указать переменные `BOT_TOKEN`, `MANAGER_CHAT_ID`
5. **Settings → Networking → Generate Domain**, указать порт `3000`
6. Скопировать выданный домен в переменную `WEBAPP_URL`, передеплоить
7. В @BotFather: `/setmenubutton` → указать домен как Web App кнопку

Таблицы в базе создаются автоматически при первом запуске.

| Команда | Действие |
|---|---|
| `/start` | Запуск бота, принятие соглашения, открытие Mini App |

## Логика обработки заявок

Все операции создают запись в таблице `orders` со статусом `pending`. Менеджеру в Telegram приходит уведомление с кнопками **✅ Выполнено** / **❌ Отклонить**.

| Тип заявки | При ✅ Выполнено | При ❌ Отклонено |
|---|---|---|
| Продажа USDT → ₽ (`exchange`) | Баланс уже списан при создании заявки | USDT возвращаются на баланс |
| Покупка USDT за ₽ (`buy`) | USDT отправлены вручную, баланс не трогается | Ничего (оплата ещё не была произведена) |
| Пополнение чеком (`deposit`, crypto) | USDT зачисляются на баланс | — |
| Пополнение SIM/QR (`deposit`, fiat) | Баланс не зачисляется (оплата фиатная) | — |
| Вывод USDT (`withdrawal`) | Баланс уже списан, чек отправлен в ЛС | USDT возвращаются на баланс |

## Безопасность

- Все запросы к API проверяются через `initData` — подпись Telegram WebApp (HMAC-SHA256), что исключает подделку запросов от чужого имени
- Баланс пользователя изменяется только на сервере, фронтенд не может инициировать начисление
- Доступ к админ-эндпоинтам ограничен по `MANAGER_CHAT_ID`
- Отзыв можно оставить только один раз на одну завершённую сделку (`UNIQUE(order_id)` в БД)
